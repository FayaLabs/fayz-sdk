## Recommended minimum implementation path

Minimum path should stay in the base scaffold and avoid prompt rewrites until Boris reviews.

Add generated-project defaults:

- dependency on `@fayz/runtime` or workspace-compatible SDK package path;
- `app.manifest.json` as the canonical starter manifest;
- `src/plugins.generated.ts` for plugin imports/registry entries generated by Fayz;
- `src/registry.tsx` for local component/action/provider registrations;
- project-level `AGENTS.md` explaining how agents should use Fayz SDK safely.

```txt

src/
  main.tsx
  registry.tsx
  plugins.generated.ts
AGENTS.md

```

## Starter manifest location

Recommended v1 location:

```txt
app.manifest.json
```

Reason:

- obvious to agents;
- JSON-serializable;
- easy for Fayz to diff, mutate, and validate;
- not tied to React code.

## Runtime imports needed

Likely minimum app entry should import runtime rendering and local registry:

```ts
import manifest from '../app.manifest.json'
import { renderApp } from '@fayz/runtime'
import { registry } from './registry'
import { plugins } from './plugins.generated'
```

Exact API depends on the final runtime lock.

## Agent guide location

Recommended:

```txt
AGENTS.md
```

It should teach:

- edit `app.manifest.json` first for pages/entities/plugins;
- use `registry.tsx` for custom components/actions/providers;
- use `plugins.generated.ts` for installed modules/plugins;
- do not fork SDK internals;
- run manifest validation/build before claiming done;
- migrations are explicit and never silently applied from UI render.

## Migration command expectations

Do not let generated apps auto-run migrations on render.

Recommended future pattern:

- SDK/plugin declares migration metadata;
- Fayz/host inspects and approves;
- project exposes explicit migration command or Fayz API applies it through a controlled path.

## Risks with existing generated apps

- `package.json` mutation is already a sensitive area with tests; SDK insertion must be non-clobbering.
- If agents are taught multiple equivalent entry paths, generated projects will diverge quickly.
- Prompt changes touch Boris-owned `packages/ai*`; route structural prompt work to Boris/review.
- Generated apps should not become dependent on a local unpublished SDK path unless deployment story is clear.
